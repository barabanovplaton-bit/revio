# Revio — План проекта

## Что это
SaaS для фрилансеров: загружаешь макеты, отправляешь ссылку клиенту, клиент ставит метки на картинках и пишет комментарии. Фрилансер видит все правки в одном месте.

## Текущий статус: Доработка логики жизненного цикла

---

## Жизненный цикл проекта (УТВЕРЖДЕНО)

```
Создание → Загрузка пакета #1 → Ссылка активна → Клиент правит →
Пакет правок #1 → Уведомление → Фрилансер смотрит →
Правит → Загружает пакет #2 → Клиент видит новые картинки →
Клиент правит → Пакет правок #2 → ... → Раунды кончились → exhausted
```

### Детали:

1. **Создание** (визард 3 шага)
   - Шаг 1: Название проекта
   - Шаг 2: Описание, имя клиента, контакт клиента
   - Шаг 3: Круги правок (free: 1-5, pro: 1-999)
   - Нумерация пакетов и правок начинается с 1
   - Статус: `waiting_for_images`

2. **Загрузка пакета #1**
   - Пустое состояние: большой dropzone (пунктирная рамка)
   - Кликаешь → file picker → файлы появляются как миниатюры
   - Порядок = порядок выбора. Можно менять перетаскиванием (точки- drag handle)
   - У каждой миниатюры: кнопка удаления (мусорка) + кнопка просмотра на весь экран (лупа)
   - Внизу: кнопка «Загрузить пакет»
   - Кликаешь → **модалка подтверждения**: «Вы уверены? Изображения будут заморожены и не смогут быть изменены»
   - Подтверждаешь → загрузка в Cloudinary → заморозка → ссылка становится активной
   - Статус: `in_progress`

3. **Клиент открывает ссылку** (`/review/[id]`)
   - Видит имя проекта, имя фрилансера
   - Видит актуальный пакет изображений
   - Может посмотреть предыдущие пакеты (кнопка «Предыдущие пакеты» → выпадающий список → кликаешь → видишь старые картинки + свои метки, read-only)
   - Ставит метки (точечные + общие комментарии)
   - Жмёт «Отправить правки» → модалка подтверждения → отправляет

4. **После отправки клиентом**
   - Статус клиента: «Правки отправлены. Ожидайте следующего пакета»
   - `isLocked = true`, `roundsLeft--`
   - Фрилансеру приходит уведомление (колокольчик + push)

5. **Фрилансер видит пакет правок**
   - В проекте: секция «Пакеты правок» — список по кругам
   - Клик на пакет →展开 список меток этого круга
   - Фрилансер смотрит, правит работу

6. **Загрузка пакета #2**
   - Фрилансер нажимает «Загрузить новый пакет»
   - Та же процедура: файлы → миниатюры → модалка → заморозка
   - Ссылка та же, но клиент при обновлении видит новые картинки
   - Реалтайм: страница клиента обновляется автоматически (onSnapshot)

7. **Повторяется** пока круги не закончатся

8. **Раунды закончились** → `exhausted`
   - Клиент: «Лимит правок исчерпан. Свяжитесь с фрилансером.»
   - Фрилансер: видит все пакеты проекта и все пакеты правок

---

## Пакеты — нумерация и терминология

- **Пакет проекта #1, #2, #3** — загрузка файлов фрилансером
  - #1 = начальные макеты (старт проекта)
  - #2 = обновление после первой правки клиента
  - #3 = обновление после второй правки
  - и т.д.
- **Пакет правок #1, #2, #3** — отправка меток клиентом
  - #1 = первая обратная связь
  - #2 = вторая обратная связь
  - и т.д.
- Нумерация начинается с 1

---

## Заморозка изображений

- После загрузки пакета картинки **замораживаются**
- Нельзя добавить, удалить, заменить
- Если фрилансер хочет обновить макеты → загружает НОВЫЙ пакет
- Старые пакеты остаются в базе (read-only)

---

## Блокировка от повторного использования

- Free: макс. 5 кругов. Pro: макс. 999 кругов
- В настройках можно **только добавлять** круги (увеличивать roundsTotal)
- Можно уменьшать ТОЛЬКО до числа уже использованных кругов
  - Пример: было 10, использовано 3 → можно уменьшить до 3, но не ниже
- Один проект = одна ссылка (`/review/[id]`)
- Если фрилансер хочет начать заново → создаёт новый проект

---

## Просмотр предыдущих пакетов (клиент)

- На странице клиента: кнопка «Предыдущие пакеты»
- Выпадающий список: «Пакет #1 — 3 метки», «Пакет #2 — 1 метка»
- Клик → показывает старые картинки + старые метки (read-only)
- Актуальный пакет показывается по умолчанию

---

## Реалтайм обновления

- Страница клиента обновляется **автоматически** через onSnapshot
- Когда фрилансер загружает новый пакет → клиент видит без обновления страницы
- Когда клиент отправляет правки → фрилансер видит уведомление мгновенно

---

## Страницы (УТВЕРЖДЕНО)

### `/` — Главная (список проектов)
- Поиск, фильтрация
- Карточки: название, описание/клиент, статус
- Меню: переименовать, закрепить, архив, удалить
- Создание нового проекта

### `/project/[id]` — Детали проекта (freelancer view)
- **Пустое состояние** → dropzone для загрузки первого пакета
- **Есть картинки** → грид картинок + «Пакеты правок»
- **Клиент отправил** → уведомление, кнопка «Загрузить новый пакет»
- **Все круги исчерпаны** → надпись
- Шапка: назад + название + шестерёнка (настройки)

### `/project/[id]/settings` — Настройки проекта
- Название, описание, клиент, контакт
- Круги правок (+/- , нельзя ниже использованных)
- Архив / Удалить

### `/review/[id]` — Страница клиента
- Показывает имя проекта и имя фрилансера
- **waiting_for_images** → «Макеты ещё не загружены»
- **in_progress + !isLocked** → MarkerCanvas + «Отправить правки»
- **in_progress + isLocked** → «Правки отправлены. Ожидайте»
- **exhausted** → «Лимит правок исchooserан»
- Кнопка «Предыдущие пакеты» (если есть что показать)

### `/login` — Авторизация (Google)
### `/notifications` — Уведомления
### `/pricing` — Тарифы
### `/settings` — Глобальные настройки профиля

## Архитектура данных

### ProjectPackage (вложенный тип)
| Поле | Тип | Описание |
|------|-----|----------|
| `round` | number | Номер круга |
| `imageUrls` | string[] | URL картинок этого пакета |
| `createdAt` | Timestamp | Дата загрузки |

### Project (Firestore: `projects`)
| Поле | Тип | Описание |
|------|-----|----------|
| `ownerUid` | string | UID фрилансера |
| `name` | string | Название проекта |
| `description` | string | Описание |
| `clientName` | string | Имя клиента |
| `clientContact` | string | Контакт клиента |
| `imageUrls` | string[] | URL картинок текущего пакета (Cloudinary) |
| `packageHistory` | ProjectPackage[] | История загруженных пакетов (старые) |
| `currentRound` | number | Текущий круг (начинается с 1) |
| `roundsTotal` | number | Всего кругов |
| `roundsLeft` | number | Осталось кругов |
| `limitMessage` | string | Сообщение при исчерпании |
| `isLocked` | boolean | Клиент отправил правки |
| `pinned` | boolean | Закреплён |
| `archived` | boolean | В архиве |
| `status` | enum | `waiting_for_images` / `in_progress` / `exhausted` |
| `createdAt` | Timestamp | Дата создания |
| `updatedAt` | Timestamp | Дата обновления |

### Marker (Firestore: `markers`)
| Поле | Тип | Описание |
|------|-----|----------|
| `projectId` | string | ID проекта |
| `round` | number | Номер круга правок |
| `type` | `point` / `general` | Точечная метка или комментарий |
| `x?` | number | Позиция X (0-1, relative) |
| `y?` | number | Позиция Y (0-1, relative) |
| `text` | string | Текст правки |
| `createdAt` | Timestamp | Дата создания |

### Notification (Firestore: `notifications`)
| Поле | Тип | Описание |
|------|-----|----------|
| `ownerUid` | string | UID фрилансера |
| `projectId` | string | ID проекта |
| `projectName` | string | Название проекта |
| `type` | enum | `revisions_submitted` / `project_completed` |
| `message` | string | Текст уведомления |
| `read` | boolean | Прочитано |
| `createdAt` | Timestamp | Дата создания |

### UserProfile (Firestore: `users`)
| Поле | Тип | Описание |
|------|-----|----------|
| `displayName` | string | Имя |
| `email` | string | Email |
| `plan` | `free` / `pro` | Тариф |
| `onboardingCompleted` | boolean | Прошёл онбординг |
